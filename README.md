# PRD · 鸟类科普养成游戏

> **项目代号**：Hack Doooooog 舰队 · 鸟类科普养成小游戏
> **文档版本**：v1.1（MVP 版）
> **最后更新**：2026-07-11
> **平台形态**：微信小程序（原生框架，微信一键登录，云端存储）

---

## 1. 产品定义

### 1.1 背景与机会

传统鸟类科普手段（图鉴、展板、讲座）传播力有限，参与门槛高、趣味性弱。游戏化（Gamification）能够将枯燥的鸟类知识转化为"养成 + 答题 + 收集"的互动循环，从而：

- **降低参与门槛**：无需先验知识，任何人都能在 10 秒内开始；
- **提升传播力**：养成与收集机制天然具备分享与炫耀属性；
- **增强粘性**：积分与成长激励形成持续学习动力；
- **播种环保意识**：在轻松体验中建立对鸟类生态的认知与情感连接。

### 1.2 产品定位

一款以 **"养鸟宠物 + 积分成长体系"** 为核心机制的鸟类科普小游戏。玩家在照料虚拟鸟、解锁知识卡片、积累积分升级的过程中，潜移默化地学习鸟类知识。

### 1.3 产品目标

| 维度 | MVP 目标 |
|------|----------|
| 核心闭环 | 答题学鸟 → 攒积分 → 喂养宠物鸟升级 → 点亮图鉴 → 复习得额外积分 |
| 内容规模 | 15 种鸟（深圳常见鸟起步），5 大知识维度 |
| 技术形态 | 微信小程序 + 云开发，微信一键登录，云端持久化，支持跨设备同步 |
| 留存 | 通过"养成进度 + 图鉴收集"形成每日回访钩子 |

### 1.4 用户画像

| 画像 | 特征 | 诉求 |
|------|------|------|
| 自然爱好者 | 喜欢观鸟、户外，但缺乏系统知识 | 轻量学习 + 收集成就感 |
| 亲子家庭 | 家长带孩子认识自然 | 寓教于乐、亲子互动 |
| 泛娱乐玩家 | 玩过养成类游戏，对科普无强需求 | 轻松收集 + 升级反馈 |

### 1.5 核心循环

```
┌─────────────────────────────────────────────────────────┐
│                                                           │
│   [答题学鸟] ──积分──> [喂养宠物鸟] ──升级──> [成长阶段]  │
│       │                        ▲                          │
│       └──[学过的鸟]──> [图鉴] ──复习积分──┘               │
│                    │                                      │
│              [知识库搜索/识别] ──选择学习──┘               │
└─────────────────────────────────────────────────────────┘
```

## 2. 功能规格

### 2.1 知识卡答题学鸟

**目标**：通过答题将鸟类知识转化为积分与收集进度。

**内容结构**：每只鸟按 **5 个知识维度** 组织知识卡，每维度对应若干题目：

1. 外形（体色、体型、辨识要点）
2. 名字（学名、俗名、英文名）
3. 食性（食谱、取食方式）
4. 环境（生境类型、分布区域）
5. 习性（繁殖、迁徙、社群行为等）

> 注：5 个维度作为知识卡内容模板固定字段，后续内容生产按模板统一录入，保证标准一致。

**答题规则**：

- 每答对 1 题获得对应积分（见 §4.1）；
- 完成某只鸟全部维度学习后，该鸟自动 **点亮图鉴**；
- 答题模式：从 10 题题库中随机抽取，答对 5 题即通过；答错一题即标记失败，需答完全部题目后判定；
- 支持三种学习入口：
  - **知识库选鸟**：从"深圳常见鸟"列表中选择；
  - **随机鸟**：系统随机推送一只未学/待复习的鸟；
  - **拍的鸟**：用户上传/拍照，调用识别能力匹配知识库（见 §2.5）。

### 2.2 养小鸟宠物

**目标**：用积分驱动养成反馈，形成回访粘性。

**成长模型（线性 5 阶段）**：

| 阶段 | 解锁条件 | 视觉表现 |
|------|----------|----------|
| 鸟蛋 | 初始 | 蛋形态，可轻触互动 |
| 幼年 | 累计喂食 N 次 / 达经验阈值 | 绒毛小鸟 |
| 成年 | 进阶经验阈值 | 完整羽色 |
| 盛年 | 进阶经验阈值 | 略带沧桑羽色 |
| 究极 | 达到满级经验（1350） | 光环/勋章，可"放归自然" |

**规则**：

- 用积分在 **饲料商店** 购买饲料；
- 喂食消耗饲料并增加宠物鸟经验值；
- 经验值跨过阈值即进入下一阶段；
- 宠物鸟达到满级后，可开启 **下一只鸟**（重新从鸟蛋开始），已养成的鸟进入"我的鸟舍"收藏。

### 2.3 个人图鉴

**目标**：沉淀学习成果，复习驱动二次学习。

**规则**：

- 展示所有已学过的鸟（按知识维度完成度标记）；
- 点击任意已点亮鸟类可进入 **复习模式**；
- 复习答对可获得 **额外积分**（+20 分，见 §4.1）；
- 未到复习时间也允许答题练习，但不加积分、不记录进度；
- 支持按"已精通 / 待巩固 / 未学习"筛选。

### 2.4 知识库浏览与搜索

**目标**：主动探索式学习入口。

**规则**：

- 内容池为"深圳常见鸟"；
- 支持 **关键词搜索**（鸟名、外形特征词、生境词等）；
- 搜索结果点击即进入该鸟知识卡学习流程（复用 §2.1）；
- 特征搜索：输入"红色、水边、大"等特征词，返回候选鸟列表（依赖特征检索能力，见 §2.5）。

### 2.5 识别与搜索能力（可插拔）

> 拍照识别与特征搜索为 **可插拔能力**，PRD 定义产品能力，不绑定具体实现。当前 API 待定。

- **拍照识别**：用户拍照 → 调用鸟类识别 API → 返回候选鸟 → 命中后进入学习；
- **特征搜索**：用户描述特征 → 调用检索/匹配 API → 返回候选鸟列表；
- **接入点预留**：本地知识库 JSON + 远程识别服务双数据源，识别结果统一映射为内部 Bird 实体。

### 2.6 新手教程

**目标**：新用户首次进入小程序时，通过引导式教程快速理解核心玩法，降低流失率。

**触发条件**：
- 新用户（`tutorialCompleted: false`）首次进入首页自动触发；
- 老用户不触发，教程完成状态存储于云端。

**教程流程（3～4 步引导）**：

| 步骤 | 内容 | 交互 |
|------|------|------|
| 1 | 领养一颗鸟蛋 | 点击鸟蛋，弹出宠物鸟介绍，确认领养 |
| 2 | 去答题学鸟得积分 | 箭头/高亮引导至"答题学鸟"入口，引导完成 1 道题 |
| 3 | 用积分买饲料喂鸟 | 引导至饲料商店，赠送首份饲料，完成首次喂食 |
| 4 | 完成！ | 展示"你已掌握基本玩法"，关闭教程，正常进入首页 |

**设计原则**：
- 采用遮罩 + 高亮 + 文字气泡的引导形式，不强退；
- 每步必做，但可在任意步骤跳过（"稍后再说"）；
- 教程中给予初始资源赠送（1 颗鸟蛋 + 1 份饲料），确保首体验流畅。

## 3. MVP 范围界定

### 3.1 包含（MVP）

- 知识卡答题（5 维度模板），15 种鸟内容；
- 积分获取：答题 + 图鉴复习；
- 养鸟宠物：买饲料 → 喂食 → 5 阶段成长 → 满级换新；
- 个人图鉴：展示 + 复习得额外积分；
- 知识库：深圳常见鸟列表 + 关键词搜索；
- 新手教程：新用户 3~4 步引导，支持跳过；
- 微信一键登录：用户点击授权按钮即可自动登录，无需手动输入，`wx.login` 获取 openid + 云开发数据库存储；
- 数据云端持久化，换设备后进度不丢失。

### 3.2 暂不包含（非 MVP / 后续迭代）

- 拍照识别与特征搜索的实装（仅预留接入点）；
- 社交排名、好友鸟舍；
- 超过 15 种鸟的扩展内容；
- 音视频叫声播放（可后续补）；
- 成就系统、每日任务、连续签到。

## 4. 数值与数据模型

### 4.1 积分模型（示例区间，开发中可调优）

| 行为 | 积分 | 说明 |
|------|------|------|
| 首次测验通过（答对5题） | +15 | 点亮图鉴基础奖励 |
| 图鉴复习通过 | +20 | 延后复习激励（高于首次学习） |
| 答错/早复习 | 0 | 不扣分，但复习间隔重置/不记录进度 |

### 4.2 成长模型（5 阶段）

| 阶段 | 所需累计经验 | 约需喂食次数（单次 +20 经验） |
|------|--------------|-------------------------------|
| 鸟蛋 → 幼年 | 100 | 5 |
| 幼年 → 成年 | 300 | 10（累计 15） |
| 成年 → 盛年 | 600 | 15（累计 30） |
| 盛年 → 究极 | 1350 | 20（累计 50） |

饲料价格：基础饲料 20 积分 / 份（MVP 单一饲料类型）。

### 4.3 数据实体（云端存储结构）

> 使用微信云开发数据库（文档型），用户数据按 `openid` 隔离。

```jsonc
// 全局用户状态（users 集合，_id = openid）
{
  "_id": "openid_xxx",
  "tutorialCompleted": false,
  "totalScore": 0,
  "currentBird": { /* 见 BirdPet */ },
  "birdShed": [ /* 已养成的 BirdPet 列表 */ ],
  "learnedBirdIds": ["bird_001", "bird_002"]
}

// 宠物鸟 BirdPet
{
  "birdId": "bird_001",
  "stage": "egg|chick|adult|elder|max", // 5 阶段
  "exp": 0,
  "feedCount": 0,
  "isRetired": false
}

// 知识库 Bird（静态内容）
{
  "id": "bird_001",
  "name": "白头鹎",
  "cover": "/assets/birds/001.png",
  "knowledge": {
    "appearance": "...",
    "name": "...",
    "diet": "...",
    "habitat": "...",
    "behavior": "..."
  },
  "quiz": [ /* 每维度题目，含题干/选项/答案/积分 */ ]
}

// 图鉴条目 CodexEntry
{
  "birdId": "bird_001",
  "learnedDimensions": ["appearance","habitat", ...],
  "learned": true,           // 是否完成首次学习测验
  "mastered": false,         // 是否真正精通（走完 4 次间隔复习）
  "progress": 1,             // 当前进度：1~5（1=刚学完，5=精通）
  "lastReviewAt": 0,
  "nextReviewAt": 0          // 下次可复习时间戳（0=无需/已精通）
}
```

## 5. 视觉与界面设计规范

### 5.1 设计风格

- **基调**：自然、治愈、清新。圆角卡片 + 柔和投影，弱拟物插画风。
- **字体**：系统默认无衬线，正文 28rpx 起，标题加粗。
- **图标**：Tab 栏、搜索框、登录标识等已替换为 `images/icons/` 下的手绘风格 PNG 图片；鸟类采用卡通手绘风。
- **动效**：喂食、升级、点亮图鉴均有轻量微动画（弹性缩放 / 飘心）。

### 5.2 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色 Primary | `#4CAF82`（森绿） | 主按钮、进度、品牌 |
| 辅色 Secondary | `#FFC857`（暖黄） | 积分、奖励、高亮 |
| 背景 BG | `#F4F8F4`（浅雾绿） | 页面底色 |
| 卡片 Card | `#FFFFFF` | 卡片底 |
| 文字 Text | `#2E3A33`（深墨绿） | 正文 |
| 次要文字 | `#8A9A90` | 辅助说明 |
| 警示 Accent | `#E8705B`（珊瑚红） | 错误、低血量提醒 |

注意⚠️，千万不能用紫色或渐变色

> 配色呼应"自然 / 鸟类"主题，主绿传递生态感，暖黄强化积分与成就感。

### 5.3 关键页面视觉说明

| 页面 | 视觉重点 |
|------|----------|
| 首页 | 顶部宠物鸟展示（当前阶段插画 + 进度条），中部积分与快捷入口（答题/图鉴/知识库），底部每日推荐卡片（点击直接弹知识卡片覆盖层） |
| 知识卡 | 顶部鸟插画 + 名称，中部维度 Tab（5 个），底部答题区；答对飘 "+10" 积分动效 |
| 宠物养成 | 全屏背景图（按鸟类动态切换），中央大图鸟（按阶段切换素材），底部经验卡片 + 喂食卡片；顶部/底部背景透明 |
| 图鉴 | 网格瀑布流，已学鸟亮色 + 完成度角标，未学鸟灰剪影；点击进入复习 |
| 知识库 | 搜索框 + 关键词标签，下列鸟卡片（缩略图 + 名称 + 一句话特征） |
| 新手教程 | 遮罩层 + 目标元素高亮 + 气泡文案 + 「下一步/跳过」按钮 |

### 5.4 界面资源清单（MVP 需产出）

- 15 种鸟 × 5 阶段 = 75 张宠物鸟插画；
- 知识卡、图鉴、商店等通用组件图标一套；
- 空状态（未学鸟剪影）、加载、升级动效素材。

## 6. 信息架构与页面流程

### 6.1 信息架构

```
首页 Home
├── 答题学鸟 Learn
│    ├── 选鸟（知识库 / 随机 / 拍照）
│    └── 知识卡答题（5 维度）
├── 宠物养成 Pet
│    ├── 当前鸟状态 + 喂食
│    └── 饲料商店
├── 图鉴 Codex
│    ├── 鸟列表（已学/待巩固/未学）
│    └── 复习模式
└── 知识库 Library
     ├── 搜索
     └── 鸟卡片 → 学习
```

### 6.2 主流程

1. 新用户首次进入 → 触发新手教程（领蛋 → 答题 → 买饲料喂食 → 完成）；
2. 老用户进入首页 → 看到宠物鸟 + 积分；
3. 点"每日推荐" → 直接弹知识卡片 → 点击"开始答题"进入 quiz 页面；
4. 点"答题学鸟" → 选鸟 → 题库答题（答对 5 题通过，答错即失败）；
5. 用积分在商店买饲料 → 回宠物页喂食 → 经验增长 → 阶段升级（鸟蛋→幼年→成年→盛年→究极）；
6. 进入图鉴复习已学鸟 → 额外积分；
7. 满级后开启下一只鸟（优先未拥有品种），循环。

## 7. 非功能需求

- **云端同步**：用户游戏数据（积分、宠物进度、图鉴）存储在微信云开发数据库，以 `openid` 隔离用户；
- **离线容错**：网络异常时使用本地缓存兜底，恢复网络后自动同步到云端；
- **性能**：启动 < 1.5s，答题交互反馈 < 100ms，云端读写 < 500ms；
- **兼容**：微信基础库 2.19+，覆盖主流机型；
- **安全**：通过云函数校验积分变更，防止客户端篡改；
- **无障碍**：正文对比度 ≥ 4.5:1，关键操作有文字标签。
